תקציר תולדות הנתונים (מ-חלק א')

לפני שאנחנו צוללים לסימולטורים, נוודא שכולנו יושבים חזק על היסודות:

  • אקסל נשבר כשמדובר באפליקציות חיות. אנחנו צריכים מסד נתונים (Database) בענן.
  • מסד נתונים מורכב מטבלאות. לכל עמודה בטבלה יש סוג נתון (INT למספר, VARCHAR לטקסט).
  • בטבלה חייב להיות מפתח ראשי (PK) - ה-ID המיוחד של השורה (הת.ז).
  • טבלאות מדברות אחת עם השנייה בעזרת מפתח זר (FK) כדי למנוע כפילויות מידע.

האתגר עכשיו: באקסל אנחנו פשוט מזיזים את העכבר ומקלידים. בענן - למסד הנתונים אין "מסך הקלדה". הוא מבין רק פקודות טקסט. איך אנחנו מבקשים ממנו לייצר את הטבלה?

1

שפת SQL: איך מדברים עם המחשב?

SQL (Structured Query Language) היא השפה שבה אנחנו נותנים הוראות למסד הנתונים. אל תיבהלו מהמילה "קוד" - זו שפה פשוטה שנשמעת כמעט כמו אנגלית.

יש 3 פקודות בסיס שחובה להכיר כדי לבנות אפליקציה. (חשוב: בעולם המודרני, ה-AI יכתוב את הפקודות האלו, אבל אנחנו חייבים לדעת לקרוא אותן כדי לאשר ל-AI את העבודה).

פקודה 1: "תבנה לי תבנית" (CREATE TABLE)

זו הפעולה של האדריכל. לפני שאפשר להכניס נתונים, צריך ליצור את ה"קופסה" הריקה ולהגדיר לה את העמודות והסוגים.

CREATE TABLE users (
  id INT PRIMARY KEY, -- תעודת הזהות (מספר)
  name VARCHAR(50) -- שם הלקוח (טקסט עד 50 תווים)
);

* התוצאה: המחשב מייצר טבלה ריקה לגמרי בשם `users` שמחכה לנתונים.

פקודה 2: "תכניס נתון פנימה" (INSERT INTO)

זו הפעולה של הפקיד או האפליקציה. יש לנו טבלה ריקה, ולקוח הרגע נרשם לאתר. המערכת שולחת למחשב פקודה לדחוף את השורה החדשה לטבלה.

INSERT INTO users (id, name)
VALUES (1, 'Yossi');

* חשוב: שימו לב שהשם 'Yossi' עטוף במירכאות בודדות. המחשב חייב מירכאות כדי להבין שזה טקסט ולא פקודה.

פקודה 3: "תציג לי נתונים" (SELECT)

הפעולה של מנתח הנתונים (Data Analyst). הטבלה מלאה, ואנחנו רק רוצים לשלוף ולראות מידע ספציפי. פקודה זו לא משנה או מוחקת כלום במסד.

SELECT *
FROM users
WHERE name = 'Yossi';

* תרגום לעברית: תבחר (SELECT) את כל העמודות (*) מתוך (FROM) טבלת 'users', איפה ש(WHERE) השם הוא 'Yossi'.

2

התנסות בענן: מחנה אימונים SQL

כשתעבדו בסביבת ענן אמיתית (כמו Google BigQuery), המסך שלכם ייראה בדיוק ככה: חלון כהה שבו כותבים את הפקודה, וכפתור "Run" שמריץ אותה מול השרתים.

הכנו לכם 3 משימות. אל תפחדו לטעות - אם תקלידו משהו שגוי, המערכת תתן לכם הערה באדום. נסו לתקן!

משימה 1: יצירת תבנית הטבלה (CREATE)

המטרה: להורות למחשב ליצור טבלה ריקה בשם students. הטבלה צריכה להכיל מזהה מסוג מספר שהוא מפתח ראשי (INT PRIMARY KEY), ושם מסוג טקסט (VARCHAR).

* השלמנו לכם את ההתחלה, נסו לסיים את הפקודה בין הסוגריים ולחצו על הרץ:

⚡ Terminal 1: Schema Builder

מה בדיוק כל מילה אומרת פה?

  • CREATE TABLE students - "מחשב יקר, תיצור לי טבלה חדשה וקרא לה students".
  • id INT PRIMARY KEY - "העמודה הראשונה תקרא id. היא תכיל מספרים שלמים (INT), והיא תהיה המפתח הראשי של הטבלה (אסור שיהיו שניים זהים)".
  • name VARCHAR - "העמודה השנייה תקרא name, והיא תכיל טקסט".

הערה: אחרי שלחצתם על "הרץ" המחשב לא הציג כלום על המסך, אבל מאחורי הקלעים בענן הוא ייצר את התבנית הריקה הזו, שמחכה לנתונים:

id 🔑name
(ריק)(ריק)
❓
בדיקת הבנה: CREATE
למה אנחנו חייבים לציין את המילים INT או VARCHAR ליד כל עמודה שאנחנו יוצרים?
אכדי להגדיר למחשב את ה"תבנית" (סוג הנתון) שהוא יקבל, וכך למנוע שגיאות של הזנת טקסט לתוך עמודה שאמורה להיות מחיר.
בכי המחשב קורא רק אנגלית.
גזה לא חובה. המחשב מספיק חכם לנחש לבד את סוג הנתונים.

משימה 2: הכנסת נתונים (INSERT) למערכת

הטבלה `students` שיצרתם במשימה 1 מחכה לנתונים. בואו נוסיף לתוכה סטודנטים!

המשימה שלכם למחזור חיים מלא:
1. לחצו "הרץ" כדי להכניס את הסטודנט מספר 1 ('Tal').
2. שנו בחלון השחור את המספר ל-2 ואת השם ל-'Dan' והריצו שוב!
3. שנו את המספר ל-3 ואת השם ל-'Shir' והריצו שוב!
ראו איך הטבלה שלכם נבנית ומתמלאת למטה בזמן אמת.

⚠️ חוק ברזל (מירכאות): מילים וטקסט ב-SQL חייבים להיות עטופים במירכאות בודדות (למשל 'Tal'). אם לא נשים מירכאות, המחשב יחשוב ש-Tal זו פקודת מערכת ויקרוס!

⚡ Terminal 2: Data Ingestion
הטבלה ריקה. לחצו על "הרץ" כדי להכניס את השורה הראשונה.
⚡ נסו "לשבור" את המערכת (אתגר המפתח הראשי):
אחרי שהכנסתם את 'Tal' (עם ID מספר 1), נסו להכניס סטודנט נוסף בשם 'Ron' אבל השאירו לו גם כן את ה-ID מספר 1. הריצו את הקוד. מה קרה? המחשב צעק! למה? כי הגדרנו את ID כ-PRIMARY KEY (מפתח ראשי). אסור שיהיו שתי שורות עם אותו מזהה! ככה מסד נתונים מגן עלינו מכפילויות לעומת אקסל.
❓
בדיקת הבנה: INSERT
מה יקרה אם תנסו להריץ את הפקודה הבאה ללא מירכאות סביב השם?
INSERT INTO students (id, name) VALUES (1, Tal);
אהמידע יכנס בהצלחה, המירכאות הן רק המלצה.
בהמחשב יעצור ויחזיר שגיאה. בלי מירכאות, המחשב יחשוב שהמילה Tal היא פקודה (כמו SELECT) ולא יבין מה אתם רוצים ממנו.
גהשם ייכנס למסד הנתונים בתור מספר במקום אותיות.

משימה 3: שליפה חכמה (SELECT)

עכשיו יש לכם טבלת students אמיתית בזיכרון, והיא מכילה את הסטודנטים שהכנסתם במשימה הקודמת (טל, דן, שיר...).

המשימה שלכם:
הקוד למטה שולף כרגע את כל הסטודנטים בטבלה. שנו את הקוד כך שישלוף רק את הסטודנט שקוראים לו 'Shir'. (רמז: תצטרכו להשתמש במילה WHERE שאחריה תנאי על העמודה name).

⚡ Terminal 3: Data Analytics
לחצו על כפתור ההרצה כדי לשלוף נתונים מהטבלה שבניתם.
❓
בדיקת הבנה: SELECT
מה יקרה אם תכתבו בסימולטור פקודת SELECT בלי המילה WHERE בכלל? (כמו: SELECT * FROM products;)
אהמחשב יחזיר שגיאה כי חובה לחפש משהו ספציפי.
בהמחשב ישלוף את כל השורות הקיימות בטבלת המוצרים (יביא את כל החנות למסך).
גהמחשב ייצור טבלה חדשה וריקה.
3

הכוח האמיתי: ה-AI כמתכנת האישי שלכם

ראיתם במחנה האימונים שהדברים מתחילים לחזור על עצמם. צריך לכתוב פקודות מדויקות, לזכור מירכאות, להגדיר VARCHAR ו-INT.

אני אגלה לכם סוד: במציאות של 2026, אנחנו כמעט ולא כותבים את הפקודות האלו בעצמנו "מאפס"!

במקום לכתוב CREATE TABLE ארוך ומייגע עם כל הסוגים והסוגריים, אנחנו הולכים ל-ChatGPT או Claude, מנחים אותם בשפה פשוטה (Prompt), והם מייצרים את ה-SQL בשנייה.

אז איפה המלכודת? (למה חברות משלמות לכם?)

הבינה המלאכותית עושה בדיוק מה שאומרים לה, אבל אין לה היגיון עסקי חכם משלה. אם לא נסביר לה איך לקשר טבלאות נכון (מפתח זר - FK), או נשכח לציין עמודה קריטית, ה-AI יכתוב קוד SQL שרץ מעולה במחשב - אבל יהרוס את האפליקציה מבחינה עסקית.

תרגיל חימום: מצאו את הטעות (הקלה) של ה-AI

ביקשנו מ-ChatGPT: "כתוב לי פקודת יצירת טבלה עבור 'מוצרים' בחנות שלנו. אני צריך מזהה למוצר, שם מוצר, ומחיר המוצר."

הנה מה שה-AI הפיק. הסתכלו טוב על שורת המחיר (price):

CREATE TABLE products (
  id INT PRIMARY KEY,
  name VARCHAR(100),
  price VARCHAR(50)
);
מה הטעות שעשה ה-AI פה?
אתגר ברמת ארכיטקט: מקימים את "ספוטיפיי" בעזרת AI
המנכ"ל אומר לכם: "אנחנו מקימים אפליקציית מוזיקה. אנחנו צריכים טבלה שתשמור את כל השירים באפליקציה".

שלב 1: הבקשה שאנחנו ניתן ל-AI (הפרומפט - Prompt):

"היי AI, אני מרים עכשיו מסד נתונים לאפליקציית מוזיקה בענן. תכתוב לי פקודת SQL שיוצרת טבלה (CREATE TABLE) בשם 'שירים'. הטבלה צריכה לכלול: מזהה שיר, שם השיר, ואורך השיר."

שלב 2: הקוד שה-AI הקיא לנו תוך שנייה:

CREATE TABLE songs (
  id INT PRIMARY KEY,
  title VARCHAR(100),
  duration_sec INT
);

הקוד טכנית נכון ועובד. אם נדביק אותו בסימולטור תיבנה טבלת שירים. אבל משהו כאן מאוד חסר בראייה עסקית. בואו נראה אם אתם מצליחים לזהות את הפאדיחה של ה-AI:

סמנו את הטעות הקריטית (מבחינת קשרים עסקיים) בקוד שה-AI כתב:

המסקנה: גישת AI-First לא אומרת "לכבות את המוח". היא אומרת שאנחנו מבינים את מושגי היסוד שלמדנו היום (PK, FK, סוגי נתונים), מנחים את ה-AI לבצע את כתיבת ה-SQL ה"שחורה", ובודקים שהוא לא פגע בארכיטקטורה שהאפליקציה צריכה כדי להצליח.

4

התמונה הגדולה: איך באמת בונים אפליקציה?

עכשיו שיש לכם את כל החתיכות של הפאזל (טבלאות, SQL, מפתחות, AI), ככה נראה מחזור הפיתוח האמיתי שאתם תבצעו בפרויקט הגמר בקורס ובמקום העבודה:

  1. איפיון (The Idea): מבינים מה האפליקציה עושה ואילו אובייקטים היא צריכה (לקוחות, הזמנות).
  2. ארכיטקטורה (ERD): קובעים אילו טבלאות ניצור, מי יהיה המפתח הראשי (PK) ואיך נקשר ביניהן (FK).
  3. הקמת התשתית: משתמשים ב-AI ליצירת פקודת CREATE TABLE או בונים אותה ידנית בכלי נוח כמו Airtable.
  4. הזרמת נתונים: משתמשים עושים פעולות באפליקציה ששולחות פקודות INSERT INTO למסד.
  5. הפקת ערך ודשבורדים: משתמשים בפקודות SELECT כדי לשלוף נתונים, לייצר גרפים ולתת למנכ"ל תובנות.

המשימה שלכם לקראת השיעור (5 דקות):

  • אתגר חשיבה: חשבו על רעיון לאפליקציה שתרצו לפתח במהלך הקורס (ניהול משמרות? ניהול סטודיו ליוגה? אפליקציית מתכונים?). נסו לרשום על דף אילו 2 טבלאות תצטרכו עבורה.
